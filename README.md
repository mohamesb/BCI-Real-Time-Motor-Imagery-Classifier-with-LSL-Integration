# Real-Time Motor Imagery Classifier with LSL Integration

A complete brain-computer interface (BCI) pipeline: raw EEG → causal artifact handling → EEGNet deep learning → real-time classification over Lab Streaming Layer.

## Quick Start

```bash
pip install -r requirements.txt
jupyter notebook realtime_motor_imagery_classifier.ipynb
```

Run all cells top-to-bottom. The dataset (BCI Competition IV 2a) downloads automatically via MOABB on first run (~420 MB).

## What It Does

| Stage | Description | Latency |
|---|---|---|
| **Data** | BCI Competition IV 2a: 9 subjects, 22 EEG channels, 250 Hz, left/right hand motor imagery | — |
| **Artifact handling** | Causal IIR bandpass (4–38 Hz), 50 Hz notch, eye blink interpolation, adaptive muscle suppression | < 1 ms |
| **Model** | EEGNet (Lawhern 2018): ~2,000 parameters, depthwise separable convolutions | < 5 ms |
| **Streaming** | LSL simulator pushes data sample-by-sample at 250 Hz; classifier pulls + buffers + infers | < 100 ms total |

## Artifact Handling Strategy

All processing is **causal** (forward-only, no future lookahead) so it runs identically in real-time:

1. **50 Hz notch filter** — IIR bandstop removes mains interference
2. **Eye blink detection** — Amplitude threshold on Fz channel; affected samples interpolated across all channels
3. **Muscle artifact suppression** — Adaptive Z-score on high-frequency power; soft suppression (×0.3) instead of rejection
4. **Bandpass filter (4–38 Hz)** — Causal Butterworth IIR preserves mu/beta rhythms while removing drift and high-freq noise

Filter states are carried across chunks for seamless streaming.

## Key Design Decisions

- **IIR over FIR filters**: Lower order = fewer operations = lower latency for same selectivity
- **Interpolation over rejection**: No gaps in the stream; classifier always gets complete windows
- **Soft suppression over hard zeroing**: Avoids transients that cause filter ringing
- **pylsl fallback**: Thread-queue mock LSL when pylsl isn't installed — identical interface, works everywhere

## Hardware Requirements

Just a computer. No EEG headset needed — the LSL simulator streams from the dataset.

To connect real hardware, replace `EEGSimulator` with a real LSL stream from OpenBCI, Muse, or any LSL-compatible device.
